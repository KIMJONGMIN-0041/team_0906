##  Rule
1. 호칭 : 반말하기 
2. 본인 작업하려고 하는거 말하고 시작하기. -> 팀장님이 조율해주기

## Git 규칙
1. 각자 브랜치 만들어 작업하기
2. main에 올릴떄는 무조건 컨펌받기 - 다같이 체크
3. push 및 pull 할때는 팀장한테 물어보기
4. commit 메모 규칙 : Hnogil/변경사항

## 프로그램 작성 규칙
1. git만 사용
2. sql은 차후에.
3. 주석달기.
4. 함수매게에 파라미터이름 같이 넣기.
5. 기본로직은 다같이 구성. / 세부프로그램은 분업.
6. python으로 간단하게 구현하기.
7. github만 사용

## 은행 계좌 프로그램 

- [이름,잔고,계좌번호]

> 이름 : 문홍일

> 계좌번호 : 000000
